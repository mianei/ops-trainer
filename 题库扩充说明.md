# 题库扩充说明

当前内置 **10 个模块 × 每模块 4 题 = 40 题**。可以无限加题，无需改代码。

## 方式一：追加文件 `scenarios-extra.json`（推荐）

在对应模块 id 下增加字符串即可，**不会覆盖**原题。

| 模块 id | 名称 |
|---------|------|
| user | 用户需求挖掘 |
| competitor | 竞品拆解 |
| feature | 功能设计推演 |
| interaction | 交互与信息架构 |
| decision | 业务决策推演 |
| data | 数据异常分析 |
| growth | 增长 / AARRR |
| retention | 用户分层 / 留存 |
| content | 文案 / 活动传播力 |
| crisis | 危机公关 |

示例：

```json
{
  "feature": [
    "你的飞书题库里的第 1 道题，粘贴成一行字符串",
    "第 2 道题……"
  ],
  "user": [
    "又一题……"
  ]
}
```

保存后 `git push`，等 Vercel 部署。页面上方会显示总题数。

## 方式二：直接改 `topics.json`

每个模块里的 `scenarios` 数组追加条目。适合大批量重写。

## 从飞书文档搬题（不能复制时）

1. **向文档所有者申请**：可复制 / 可导出 / 可创建副本到你空间  
2. **导出**：若有权限，飞书 → 导出 Word / PDF / Markdown  
3. **创建副本**：部分文档允许「创建副本」到自己的飞书，副本里往往可以复制  
4. **截图 + 识别文字**：手机相册 / 微信扫一扫 / iOS 实况文本  
5. **手动口述**：开飞书对着打，录入 `scenarios-extra.json`  
6. **分批发我**：截图或能打多少发多少，可帮你整理成 JSON  

注意：绕过版权/权限限制批量爬取可能违规，建议优先找文档所有者开权限。

## 新增一整类模块

在 `topics.json` 的 `topics` 数组里新增一项（需含 id、label、fieldLabel、placeholder、scenarios、systemPrompt、cat、catLabel、desc）。  
模块 id 若为新 id，需在 `index.html` 的展示逻辑里会自动出现（有 cat 即可分组）。
