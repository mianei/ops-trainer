# Assessment Framework

Use this as a scanning checklist when a user provides a resume, experience list, or target role. Do not output all dimensions mechanically; use them to find the best project direction and risks.

## 1. Technical Understanding

Look for:

- Concrete AI use cases, not only "used ChatGPT".
- Awareness of model limits, hallucination, context window, RAG, Agent, prompt chains, evals, cost/latency/effect tradeoffs.
- Ability to explain technical choices in product language.

Ratings:

- Strong: can explain AI chain, eval, badcase, and tradeoffs.
- Usable: has AI project/tool experience but shallow evaluation.
- Weak: only tool usage or technical terms.
- Missing: no AI signal.

## 2. Product Judgment

Look for:

- User, scenario, pain point, priority, options, and why this solution matters.
- Product value before feature list.
- Human-AI division: what AI handles, what humans confirm.

Negative signals:

- "Participated", "assisted", "built features" without user or decision.
- Feature list without product value.

## 3. AI Project Expression Quality

This is the highest-priority dimension for the skill.

A strong AI product project includes:

```text
problem definition -> solution options -> AI chain -> evaluation -> badcase -> iteration -> resume expression
```

Check:

- Is AI doing something specific?
- Why use AI instead of rules, tables, or manual process?
- Is there an eval set?
- Are badcases classified?
- Are metrics credible?
- Can the user answer interview follow-ups?

## 4. Data and Evaluation Ability

Look for:

- SQL/Python/data analysis only if relevant.
- Metric system: north-star, process, guardrail, AI quality metrics.
- AI-specific metrics: accuracy, recall, direct resolution rate, fallback/manual transfer rate, hallucination rate, answer quality score, badcase ratio.

Common issue:

- User writes "data improved" without definition, sample, or cause.

## 5. Transferability of Existing Experience

For转行 users, identify raw materials:

- customer questions
- business processes
- SOPs
- forms/tables
- research notes
- documents/FAQ
- annotation or quality inspection
- user interviews
- support tickets
- reports and evidence sources

Judge whether the experience can become:

- RAG/knowledge assistant
- intelligent customer service evaluation
- prompt/diagnosis tool
- workflow/SOP AI automation
- data dashboard/anomaly system
- Agent/Skill workflow
- prediction/recommendation/risk model

## 6. Self-Driven Evidence

Positive signals:

- side project, demo, eval set, benchmark, product teardown, AI tool comparison, public writeup.

If missing, recommend 3-7 day artifacts:

- 20-50 anonymized samples
- flowchart
- scoring rubric
- badcase table
- before/after prompt chain
- prototype screenshot
- interview answer sheet

## 7. Resume Narrative

Prioritize:

- AI project or transferable project near the top.
- Compress school-like honors and low-signal activities.
- Translate technical terms into product decision language.
- Use data only with口径.

Suggested classification:

| Category | Meaning | Action |
| --- | --- | --- |
| Deep-dive | Real material + AI intervention + eval possible | Build full project plan |
| Auxiliary proof | Supports domain/technical/product ability | Mention briefly |
| Compress/delete | Low AI signal or school-like filler | Reduce space |
| Not suitable | Requires fabrication to AI-ify | Do not package |
