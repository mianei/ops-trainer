# 题库内容审计说明

本文档整理自产品使用反馈与截图中的问题，并说明**练习题**与**知识库**如何分工。

---

## 一、反馈中暴露的主要问题

### 1. 框架/知识写成了「题干」

**现象（截图典型）：** 题干像笔记摘要，不像面试提问。

| 错误示例（知识口吻） | 应如何处理 |
|----------------------|------------|
| 真需求应能回答：解决谁的痛点… | **知识** → 知识库「真需求三问」；**练习** → 「面试官问：如何区分真需求与伪需求？」 |
| 朋友圈不放首屏：首屏应服务最高频… | **知识** → 知识库；**练习** → 「为什么朋友圈不做首屏？」 |
| 考察抗压与闭环：困难—压力点—… | **知识** → 答题结构；**练习** → 「请描述一次最大困难…」 |
| 训练：把…换成你自己业务… | 改为明确 **请…** 指令，或拆成知识+练习两栏 |

**原则：** 知识库负责「是什么、结构是什么」；练习题只负责「请你分析/回答什么」。

### 2. 陈述句冒充题目

**现象：** 一整段「A 与 B 异同：…各 1 点」但没有「请」字，用户不知道要输出什么。

**处理：** 统一改为「请对比…」「请分析…」「请各写 N 点…」。

### 3. 标签与模块混淆

**现象：** 飞书面经里的**知识块**、**框架总结**被整段贴进 `scenarios-*.json`。

**处理：**

- 框架、定义、对比表 → `knowledge.json`
- 可独立作答的 case / 面试问法 → 保留在 scenarios

### 4. 产品/UI 类问题（已单独修复）

- 访问码 404/400/500
- 移动端布局、模块选择
- 答题无记录、需一人一号（见 v4.36+ 账号与 Redis 记录）

## 二、自动扫描结果（维护用）

运行：

```bash
node scripts/audit-content.js
```

当前规则会标记：

| 类型 | 含义 |
|------|------|
| `knowledge-as-prompt` | 含「应能回答/考察/训练：」等，疑似知识当题 |
| `missing-clear-question` | 无问号且无「请/如何/分析」等指令词 |
| `colon-statement` | 「标题：长陈述。」式笔记体 |

**说明：** 含「各写/写出/列举」的题干多数仍是有效练习题，扫描结果需人工扫一眼，不必全改。

## 三、内容分工（必读）

| | 知识库 (`knowledge.json`) | 练习题 (`scenarios-*.json`) |
|--|---------------------------|-------------------------------|
| 目的 | 阅读、复习框架 | 独立作答 + AI 点评 |
| 形式 | 卡片：标题 + 正文 | 场景 + 明确提问 |
| AI | 不提交、不评分 | 点评 + 思维成长对比 |
| 示例 | AARRR 定义、危机四段式 | 「某 App 完课率 12%，你怎么分析？」 |

---

## 四、后续维护建议

1. 新增飞书/面经内容时，先判断：**这是让人背的还是让人练的？**
2. 练的 → 写清「请…」；背的 → 只进 `knowledge.json`
3. 大改后运行 `node scripts/fix-scenario-prompts.js` 与 `node scripts/audit-content.js`
4. 知识库在 App 内 **「知识库」**  Tab 查看，与训练区分离

## 五、已执行的批量修正（历史）

`scripts/fix-scenario-prompts.js` 已将 14+ 条典型框架式题干改为清晰面试问法（如 Q11 真需求、Q82 朋友圈首屏、Q5 抗压等）。对应**知识点**已迁入 `knowledge.json`，避免在练习题里重复堆定义。
